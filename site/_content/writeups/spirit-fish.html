The Spirit fish gate - fishing's rarest catch, decompiled

Legend of Legaia's fishing community has chased the Spirit fish for decades. The accepted table says Normal Lure at Buma with a “Good!” bait gives a 1-in-32 shot - yet players report hundreds of catches, whole sessions, entire evenings with nothing, and a binomial argument that the drought is statistically impossible. The folklore blames Square-button presses, pond position, cast distance. The disassembly of the fishing overlay settles it: the community table has every species in the right cell, but the rare column is not a probability at all. It is a six-condition gate, evaluated in a single instant - and most of the conditions never made it into any guide.

TL;DR
The species you hook is a table lookup: lure row × “band” column. The band roll only ever outputs 0–3; band 4 - where Spirit lives - is written by one store, behind a gate: over 50 lifetime casts (a saved counter), the counter even at that moment, Normal Lure, the third rod, band 0 active at the strike, and then a 1-in-16 roll. The “Good!” splash is real but treacherous: four button-rhythm templates all fire it, and the matched template's id becomes the band - only the one ending in a Square tap selects the rare band; the natural pump-Cross rhythm deterministically selects the most common one.

The table everyone knows - and the column the dice can't reach

The hooked species is assigned in the pre-hook half of the hooked-fish handler FUN_801d26cc: species = spawn_table[lure*8 + band]. The equipped lure (_DAT_80084450) picks the row; the cast band (DAT_801d90e8, 0–4) picks the column. Decoded from the disc (rodata 0x801D8334), the species land exactly where the community's ASCII table put them - the guide's fieldwork was excellent:

What the dice actually do

While the lure sits deep enough (line record over 500 units) and no fish is hooked, a countdown re-rolls the band: rand() & 0xfff against three fixed cutoffs. The guide's 184/48/16/8-out-of-256 estimate had the right shape; the real cutoffs are slightly harsher at the rare end. And no outcome maps to band 4 - there is no fifth cutoff.

“Good!” is real - and usually working against you

The same countdown that re-rolls the band first consults a reel-cadence recogniser (FUN_801d3db4): a 16-slot ring buffer of {button, held-frames} reel history, walked backwards against four rodata templates (0x801D87D4), each hold matched with ±10 frames of tolerance. A match does two things: it fires the splash players read as “Good!”, and it stores the matched template's id as the band. That second half is the piece the folklore never had - and it cuts both ways:

Six conditions, one instant

When a strike lands (reel held, per-frame strike roll passed, nothing hooked yet), one branch in FUN_801d26cc - executed immediately before the species lookup - decides whether an active band 0 upgrades to band 4. Everything must hold at once:

if (venue == Buma) {                          // DAT_801d90d0 == 0
    if (0x32 < casts                          // > 50 lifetime casts  (_DAT_80084460)
        && (casts & 1) == 0                   // cast counter EVEN at this instant
        && lure == 1                          // Normal Lure          (_DAT_80084450)
        && rod  == 2                          // third rod            (_DAT_80084454)
        && band == 0)                         // rare band active     (DAT_801d90e8)
        if ((rand() & 0xf) == 0)              // ... then 1 in 16
            band = 4;                         // -> Spirit
} else {                                      // Vidna
    if ((casts & 1) == 0 && lure == 2 && rod == 2 && band == 0)
        if ((rand() & 3) == 0)                // 1 in 4, no cast-count threshold
            band = 4;                         // -> Baraba
}

Three of these were invisible to players. The cast counter (_DAT_80084460) lives in the save block: it increments once per cast at the moment the flown lure lands, persists across sessions, and is saved with the game - a fresh fisher literally cannot catch Spirit inside their first 51 casts. Its parity makes every other cast silently ineligible, whatever else you do right. And the rod requirement appears in no community table at all: with anything but the third rod, Spirit is not rare - it is unreachable. The counter's parity does leak in one place: it flips which way the lure drifts when it bumps a wall, the only player-visible trace of the even/odd gate.

The community's binomial argument was sound arithmetic on the wrong model: under a flat 1/32, 250 Spiritless catches has probability ~0.04% - a genuine anomaly. Under the real gate it is the majority outcome. And the players who did land Spirit are the ones who, knowingly or not, walked the whole gate: right rod, deep cast, an even-numbered cast, and a reel rhythm that happened to end in a Square tap. Baraba, by contrast, is dramatically easier - no cast-count threshold and a 1-in-4 upgrade - which matches its reputation as merely uncommon rather than mythical.

One last twist for the optimal hunter: passing the gate only means a Spirit is on the line. Its species record carries the table's largest pull factor - the hardest tension fight in the minigame - so the gate can pass and the catch still snap off in the reel duel.

Provenance

Every mechanic above is read from the MIPS disassembly of the fishing overlay (PROT entry 0972, load base 0x801CE818) and verified against a retail USA disc image: the strike branch and band-4 gate in FUN_801d26cc, the cadence recogniser FUN_801d3db4, the reel-button decode FUN_801d7450, the species table at 0x801D81A4, the venue spawn tables at 0x801D8334 / 0x801D8434, and the gesture templates at 0x801D87D4. The full subsystem reference - state machine, tension fight, scoring, prize exchange - is minigame-fishing.md; the spawn-table and species parsers live in legaia_asset::fishing_species. No Sony-owned bytes appear here - only addresses, offsets, and decoded parameters.
