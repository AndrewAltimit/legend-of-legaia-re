Game world

Every town, dungeon, and battlefield ties back to a CDNAME scene label and a slice of the PROT.DAT archive. This page joins the curated walkthrough data (towns, enemy roster, bosses, shop counts) with the disc-side CDNAME map so you can jump from any location into the asset viewer pre-filtered to that scene.

How this page works

The cards below pull from two sources that the project already curates: the gamedata TOML files in data/gamedata/ (enemy / boss / shop tables from public walkthroughs) and the curated CDNAME map in site/_gen.py (label → PROT-block start). Click a card to deep-link into the asset viewer with the matching scene category pre-selected.

Note: walkthrough names ↔ CDNAME labels don't always match 1:1 (balden in CDNAME is the Japanese name for Vidna, geremi is Jeremi, etc.). Where the mapping is known, the card shows both. Some named locations are field maps without a settlement; those still appear here for their enemy rosters.

Towns

Visitable settlements - NPC dialog, shops, the inn, and the save crystal. Each entry maps to a town*, izumi, vell, geremi, balden, rayman, conc, station, or deene CDNAME block.

Field areas & dungeons

Non-town field maps - mountains, caves, castle interiors, dream sequences, and the world-map overworld. Most use the field-pack + scene-bundle containers.

Open questions

The walkthrough ↔ CDNAME mapping is firm for ~16 named towns. A handful of CDNAME labels (town0b / town0c / town0e, doman, retock, kor*, juui*, rugi) are field-pack scenes whose walkthrough names haven't been pinned. If you map one, the curated table in site/_gen.py (CDNAME_SCENES + TOWN_TO_SCENE) is the single source of truth.
