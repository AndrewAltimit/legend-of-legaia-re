ROM patcher (in browser)

Run the randomizer on your own Legend of Legaia disc image, entirely in your browser. Pick a preset (or fine-tune), choose a seed, and download a patched image. The disc bytes never leave your machine - the patch runs as WebAssembly locally, nothing is uploaded.

Two files download: the patched .bin and a matching .cue (same base name). Keep them together and point your emulator at the .cue - it references the .bin by name. Legend of Legaia is a single-track MODE2/2352 disc, so the cue is just a track header. If you'd rather reuse your original .cue, rename the patched .bin to match its FILE line.

  
Heads up: a full disc is ~470 MB, so patching holds a couple of copies in memory and the download is the same size - give it a moment, and use a desktop browser with enough RAM. The patched image contains Sony data: it is for your own local play, do not redistribute it. (Sharing a run is what the CLI's seed + PPF patch are for.)






  
What it does

  
This is the same randomizer / disc patcher the CLI runs, compiled to WebAssembly. Each option is a deterministic, seeded edit - same-size in place for drops / encounters / chests / steals, and a variable-length MAN rebuild for doors (the disc's total size never changes either way). Every randomizable category offers two modes: Shuffle (rebalanced - redistributes the existing set, so difficulty is preserved) or Randomize (wild - draws fresh from the whole valid pool).

  
    
    
  • Monster drops - reassigns the item each monster drops (and its chance). Shuffle redistributes the existing drops; Randomize draws from the valid item table.
    • 
    
  • Random encounters - reassigns the monster ids in each scene's encounter formations. The Monster pool sub-option sets how wide a net is cast: Scene (each area's own monsters - every swap is one the area already loads), Kingdom (any monster from the same Drake / Sebucus / Karisto kingdom), or World (any monster on the disc, so late-game monsters can appear at the start). Boss / scripted fights are never touched. The Solo strong fights sub-option keeps the wider pools fair: it scores each monster by its combat-stat budget and, for any randomized formation whose strongest member is at least twice the area's native average, drops the pack down to that lone enemy - so an out-of-area heavy hitter shows up alone, never two or three at once. The baseline is each area's own original monsters (a stand-in for the party's strength there), so it adapts as the game scales.
    • 
    
  • Treasure chests - reassigns the item each chest gives (the field-VM GIVE_ITEM inline operand), globally across every chest. The full quest / key / story item set (every unsellable item the disc prices at 0 - door keys, garden tools, eggs / talismans / books, letters, fishing rods, casino cards - minus the chest-found Ra-Seru equipment) is kept static so progression items stay where the player expects them and never spawn in an unrelated chest.
    • 
    
  • Town shops - reassigns what each town store sells (the field-VM op 0x49 stock list).
    • 
    
  • Casino prizes - reassigns the casino prize-exchange items; each prize keeps its coin price and progression gate.
    • 
    
  • Steal items - reassigns what each monster yields to the Evil God Icon (the static SCUS_942.54 steal table). The steal chance is always preserved - only the item changes.
    • 
    
  • Arts button combos - reassigns the directional combo that fires each Tactical Art (the SCUS_942.54 arts table, the one place the combo lives, so the menu arrows and the trigger always match). Every art keeps its input count and each character's combos stay unique. Each character's Miracle Art is left alone.
    • 
    
  • Monster stats - redistributes each monster's combat stats (HP / MP / ATK / DEF↑ / DEF↓ / AGL / SPD) from the battle_data archive. Shuffle permutes each stat column across the roster (the difficulty budget is kept); Randomize draws each stat from that column. Spirit/SP is left alone.
    • 
    
  • Special-attack power - redistributes the per-move power of the battle-action move table (enemy specials & Seru magic - not party Tactical Arts). Each move keeps its own animation, effects, and sound; only how hard it hits changes.
    • 
    
  • Element affinity - scrambles the 8×8 element matrix (which element is strong / weak against which). Shuffle keeps the same number of weaknesses & resistances, just between different pairs; the per-character element assignment is untouched.
    • 
    
  • Spell MP costs - redistributes the MP costs of the named, costed spells in the SCUS_942.54 spell table. Free / internal enemy-tier entries never gain a cost.
    • 
    
  • Equipment stat bonuses - redistributes the passive stat bonuses (INT / ATK / DEF↑ / DEF↓ / SPD) of the equipment table, within each slot type (weapon power lands on another weapon, armor on armor). The equip-character restriction, accessory passive, and slot type never move.
    • 
    
  • Equipment drops - a low per-battle chance (default 5%) to win one extra random weapon / armor / accessory, granted on top of the normal drop. Because a monster has only one drop slot, this can't be a data edit (that would destroy the normal drop) - instead it injects a tiny code hook into the battle-end reward routine that rolls the chance and adds a random gear id. The normal drop table (vanilla or shuffled / randomized) is never touched.
    • 
    
  • Experience on the run - banks a slice of a fight's experience into the party whenever they successfully run away (vanilla awards nothing for fleeing). Like the equipment drop, this can't be a data edit - it injects a tiny code hook into the battle-action escape teardown that sums the fled formation's listed experience, takes 5% of it, and adds that to every party member's experience. The experience shows in the status screen immediately and applies as a level the next time a won battle tallies it.
    • 
    
  • Enemy ally - a 20% chance per battle that one enemy starts the fight already in a charmed state and turns on its own side, fighting the rest of the enemies until the battle is won. This is not an added or controllable party member - retail battles are hard-wired to three party slots, so a genuine fourth combatant isn't possible. Instead it reuses the game's own Confuse/Charm status: the “AI-delegated” flag (+0x16E & 0x380) the action SM normally sets on a confused party member to flip its target to the opposite side. Setting it on an enemy at battle setup makes that enemy attack the other enemies. A code hook in battle setup sets the flag on the frontmost enemy on a roll; a companion one-word widen of the victory check (the monster-wipe gate's andi 0x4 → 0x384) keeps the charmed enemy from counting as one you still have to defeat. It fires only in multi-enemy fights: the routine reads the 2nd formation slot and skips the charm when the fight has a single enemy, because charming the lone enemy of an input-gated tutorial (the Rim Elm sparring match) or a solo boss softlocks the scripted fight - it waits for the enemy that is now an ally.
    • 
    
  • Shiny Seru - a 2% chance per battle that the frontmost capturable enemy spawns as a rare shiny variant: its combat stats are boosted +35% at battle load and it renders translucent, and the Seru you capture from it deals +35% damage on every future cast (in addition to its normal abilities), permanently. As cosmetics, the summoned creature renders semi-transparent on a shiny cast and a “+35% DMG!” caption appears one line below the native “Magic effect:” box (so they stack instead of overlapping). “Capturable” is decided from an allowlist built at patch time from the disc's own monster names - every enemy named after a player Seru-magic (Gimard, Theeder, Vera, … and their variants) - so common monsters like Gobu Gobu are never shiny. The persistent boost is stored in a spare per-spell byte in the character record (so it survives a memory-card save and the spell-list shuffle when you capture more Seru); the damage hook reads it (×135/100) and a grant-shift hook keeps it aligned as the spell list slides. All routines live in preserved reference-free SCUS_942.54 runs verified to be outside every live data table - disjoint from every other code feature so they all compose. Note: applies to Seru captured after patching - a Seru already captured on an unpatched save won't retroactively become shiny.
    • 
    
  • Weapon specialty - reassigns which weapon class each character favors (Vahn blades, Noa claws, Gala clubs/axes by default), permuting the three favored classes among them. Equipping an off-class weapon widens that character's Arms command in an arts combo, so fewer commands fit; this rewrites the per-(character, weapon) arm-cost byte in the player battle files (decompressing / re-compressing each section in place). The Astral Sword stays always-wide and can't be specialized in.
    • 
    
  • Doors / exits - reassigns where scene-transition doors lead (the field-VM 0x3F named-scene-change ops). Two-way re-pairs doors so you can walk back the way you came; one-way reassigns each independently. A few big overworld hubs whose data can't grow in place keep their original doors.
    • 
    
  • House doors (experimental) - shuffles the intra-town 0x23 MOVE_TO target tiles per scene (multiset-preserving). The op is shared with NPC/cutscene movement, so this also scrambles some actor positions within each town - opt-in chaos.
    • 
    
  • Starting items - replaces the new game's fixed Healing Leaf ×5 with distinct random consumables (each a small random count), by rewriting the new-game seed code in SCUS_942.54 (FUN_80034A6C) in place. The random fill shares the seed's seven-slot capacity (five with all-warps) with the convenience toggles below and adds on top of them rather than crowding them out.
    • 
    
  • Starting level - begins the new game with the starting party already at the chosen level (up to 14) instead of 1. The new-game seed routine stamps the displayed level on every roster slot, so this rewrites that loop to the chosen level and recomputes each growth-capable character's stats (Vahn / Noa / Gala) from the disc's own growth curves - a level-10 start gives the whole party level-10 HP/ATK, not level-1 stats under a level-10 label. Each of those characters also gets a matching cumulative-XP / next-threshold seed so every status readout is exact - not just the lead's.
    • 
    
  • Unused enemies - brings back two fully-formed monsters the shipped game never places: “Comm” and the Evil Bat. Only takes effect with Randomize encounters (a Shuffle can't introduce a new monster).
    • 
    
  • Unused items - adds two items the game ships but never hands out: “Something Good” (a sell item) and a no-name accessory the randomizer names “Seru Bell”. Only affects the Randomize drop / chest / steal fills.
    • 
  

  
The same seed always produces the same patched image, so a seed (shown after patching) reproduces a run exactly. Edits that can't be re-packed into their original footprint are skipped and reported rather than corrupting the disc.
