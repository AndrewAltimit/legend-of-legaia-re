ROM patcher (in browser)

Run the randomizer on your own Legend of Legaia disc image, entirely in your browser. Pick a preset (or fine-tune), choose a seed, and download a patched image. The disc bytes never leave your machine — the patch runs as WebAssembly locally, nothing is uploaded.

Two files download: the patched .bin and a matching .cue (same base name). Keep them together and point your emulator at the .cue — it references the .bin by name. Legend of Legaia is a single-track MODE2/2352 disc, so the cue is just a track header. If you'd rather reuse your original .cue, rename the patched .bin to match its FILE line.

  
Heads up: a full disc is ~470 MB, so patching holds a couple of copies in memory and the download is the same size — give it a moment, and use a desktop browser with enough RAM. The patched image contains Sony data: it is for your own local play, do not redistribute it. (Sharing a run is what the CLI's seed + PPF patch are for.)






  
What it does

  
This is the same randomizer / disc patcher the CLI runs, compiled to WebAssembly. Each option is a deterministic, seeded edit — same-size in place for drops / encounters / chests / steals, and a variable-length MAN rebuild for doors (the disc's total size never changes either way). Every randomizable category offers two modes: Shuffle (rebalanced — redistributes the existing set, so difficulty is preserved) or Randomize (wild — draws fresh from the whole valid pool).

  
    
    
  • Monster drops — reassigns the item each monster drops (and its chance). Shuffle redistributes the existing drops; Randomize draws from the valid item table.
    • 
    
  • Random encounters — reassigns the monster ids in each scene's encounter formations. The Monster pool sub-option sets how wide a net is cast: Scene (each area's own monsters — every swap is one the area already loads), Kingdom (any monster from the same Drake / Sebucus / Karisto kingdom), or World (any monster on the disc, so late-game monsters can appear at the start). Boss / scripted fights are never touched.
    • 
    
  • Treasure chests — reassigns the item each chest gives (the field-VM GIVE_ITEM inline operand), globally across every chest. A curated set of quest / key items (Mary's Diary, Dark Stone, the Genesis-Tree garden tools, Silver Compass, Old Rod) is kept static so progression items stay where the player expects them.
    • 
    
  • Town shops — reassigns what each town store sells (the field-VM op 0x49 stock list).
    • 
    
  • Casino prizes — reassigns the casino prize-exchange items; each prize keeps its coin price and progression gate.
    • 
    
  • Steal items — reassigns what each monster yields to the Evil God Icon (the static SCUS_942.54 steal table). The steal chance is always preserved — only the item changes.
    • 
    
  • Arts button combos — reassigns the directional combo that fires each Tactical Art (the SCUS_942.54 arts table, the one place the combo lives, so the menu arrows and the trigger always match). Every art keeps its input count and each character's combos stay unique. Each character's Miracle Art is left alone.
    • 
    
  • Monster stats — redistributes each monster's combat stats (HP / MP / ATK / DEF↑ / DEF↓ / AGL / SPD) from the battle_data archive. Shuffle permutes each stat column across the roster (the difficulty budget is kept); Randomize draws each stat from that column. Spirit/SP is left alone.
    • 
    
  • Special-attack power — redistributes the per-move power of the battle-action move table (enemy specials & Seru magic — not party Tactical Arts). Each move keeps its own animation, effects, and sound; only how hard it hits changes.
    • 
    
  • Element affinity — scrambles the 8×8 element matrix (which element is strong / weak against which). Shuffle keeps the same number of weaknesses & resistances, just between different pairs; the per-character element assignment is untouched.
    • 
    
  • Spell MP costs — redistributes the MP costs of the named, costed spells in the SCUS_942.54 spell table. Free / internal enemy-tier entries never gain a cost.
    • 
    
  • Equipment stat bonuses — redistributes the passive stat bonuses (INT / ATK / DEF↑ / DEF↓ / SPD) of the equipment table, within each slot type (weapon power lands on another weapon, armor on armor). The equip-character restriction, accessory passive, and slot type never move.
    • 
    
  • Equipment drops — turns every monster's drop into a rare random weapon / armor / accessory at a tiered chance. Owns the single drop slot, so it overrides Monster drops.
    • 
    
  • Doors / exits — reassigns where scene-transition doors lead (the field-VM 0x3F named-scene-change ops). Two-way re-pairs doors so you can walk back the way you came; one-way reassigns each independently. A few big overworld hubs whose data can't grow in place keep their original doors.
    • 
    
  • House doors (experimental) — shuffles the intra-town 0x23 MOVE_TO target tiles per scene (multiset-preserving). The op is shared with NPC/cutscene movement, so this also scrambles some actor positions within each town — opt-in chaos.
    • 
    
  • Starting items — replaces the new game's fixed Healing Leaf ×5 with up to five distinct random consumables (each a small random count), by rewriting the new-game seed code in SCUS_942.54 (FUN_80034A6C) in place.
    • 
    
  • Unused enemies — brings back two fully-formed monsters the shipped game never places: “Comm” and the Evil Bat. Only takes effect with Randomize encounters (a Shuffle can't introduce a new monster).
    • 
    
  • Unused items — adds two items the game ships but never hands out: “Something Good” (a sell item) and a no-name accessory the randomizer names “Seru Bell”. Only affects the Randomize drop / chest / steal fills.
    • 
  

  
The same seed always produces the same patched image, so a seed (shown after patching) reproduces a run exactly. Edits that can't be re-packed into their original footprint are skipped and reported rather than corrupting the disc.
