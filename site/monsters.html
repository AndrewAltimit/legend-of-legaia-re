Enemy table

Decode every enemy's stat record straight from your own disc image. The full archive — names, HP, MP, and combat stats for the whole roster — is LZS-decompressed in your browser via WebAssembly. No game data ships with this site; bring your own .bin and nothing leaves your machine.

* The first two stat columns are tentatively mapped (S0 ≈ attack, S1 ≈ defense); the remaining stat slots are still being pinned, so they're shown as raw record values. Source layout: PROT entry 0867_battle_data, one fixed 0x14000-byte LZS slot per monster id. Details on the battle subsystem page.

How it works

The stat archive is the extended footprint of PROT entry 867. Each monster id n (1-based) maps to slot (n−1)·0x14000, framed as [u32 decoded-size][Legaia LZS stream]; the decoded block's head is the stat record the battle loader copies into a live actor. The same legaia_asset::monster_archive parser used by the native tools and the engine runs here, compiled to WebAssembly.

Files are read with the browser's FileReader API directly into WebAssembly memory. Nothing is uploaded; the project ships zero Sony bytes (bring-your-own-disc). Native equivalent: asset monster-archive extracted/PROT/0867_battle_data.BIN (add --id N for a single record).
