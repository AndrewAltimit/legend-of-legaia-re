Legend of Legaia — reverse engineering & clean-room engine port

Two coordinated tracks under one repo. Track 1: extract every asset on the disc, document every format with provenance back to a Ghidra function, build round-trip parsers. Track 2: a clean-room Rust engine port that consumes the extracted data, in the same legal pattern as ScummVM, OpenRCT2, OpenMW, and OpenLara. Bring your own disc image; the toolkit handles the rest.

Where we are

The disc is fully readable. Every major subsystem is traced and documented. All three core game VMs are ported, plus the battle action state machine and effect pool. The combat-stat machinery is decoded. The asset preservation track ships round-trip parsers for every documented format.

Disc + asset preservation
Mode2/2352 reader, ISO9660 walker, PROT.DAT TOC math, format detectors classifying the bulk of PROT entries by class. Round-trip parsers ship for TIM, TMD, VAB, SEQ (both u16 and u32 BE version variants), MES (partial), ANM (container + per-bone keyframe walker), MDT, LZS, plus the scene-bundle family. PSX memory-card walker round-trips Legaia save blocks via save-tool.
Static analysis
Function dumps committed for the bulk of the binary. Closed-loop on the citation graph at >98%. All three game VMs decoded plus the battle action state machine. All major SCUS subsystems traced.
Engine port — Phase 1
Asset viewer with PSX VRAM emulation, textured TMD rendering, stage geometry, VAB playback, SEQ + VAB sequencer, dialog typewriter, field-VM scene runner with dialog rendering, battle-scene SM driver. In-browser version live.
Engine port — Phase 2
Runtime port. Actor VM (13 ops), Field VM (43 ops), Move VM (71 + 61 ext sub-ops), Effect VM, and the battle action state machine all ported. battle_formulas ships damage / MP-cost / accuracy / RNG kernels. Composite World in engine-core wires them together with a scene host, dialog panel, and BGM director. Clean-room SPU mixer + sequencer drive playback.
Engine port — Phase 3
Gameplay assembly. Field-VM scene runner ticks real per-scene event scripts and renders dialog inline. Battle-action SM drives a working scene with party + monster slots end-to-end (synthetic battle resolves to BattleEndCause via the formula damage path). Menu state machine + disk-backed save / load wired through engine-core::menu_runtime::MenuRuntime with per-slot files. The 16-arm action validator (FUN_8003FB10) is ported as engine-vm::action_validator — every UI / item-cap / status / capture-flag arm covered. The third motion VM (FUN_8003774C) is ported as engine-vm::motion_vm and feeds a runtime Camera in engine-core that follows the active actor. ANM playback driver (legaia_anm::AnimPlayer) drives per-bone keyframe interpolation each frame.
Top-level shell
The legaia-engine binary (crates/engine-shell) is the canonical "boot a scene from PROT bytes alone" entry point. Subcommands: info (one-line scene-asset summary), list-scenes (CDNAME map walker), play (tick the engine for N frames against a scene, drive the camera + BGM director + per-actor move VMs, log scene transitions), save / load (round-trip a slot file). BootSession in the same crate is the embedding API: SceneHost + Camera + AudioBgmDirector wired together; the play subcommand is a thin wrapper around it.
WASM target
legaia-engine-vm + legaia-engine-core + legaia-engine-audio all compile to wasm32-unknown-unknown. The in-browser viewer exposes a minimal LegaiaRuntime bridge for ticking the World + driving the menu state machine from JS — proof that the per-frame engine path runs in the browser. Full disc-bytes-to-Vfs integration is the follow-up.
Open work
Per-record ANM bytecode bytes for the residual non-keyframe-table records (the keyframe-table path is ported via AnimPlayer; bytecode-style records still need an overlay capture). Five gameplay overlays still uncaptured: world map, save / load screen, shop, cutscene, level-up (PROT 891). Final reverse of the per-scene DATA_FIELD slot semantics inside field-pack containers (97-slot schema understood structurally, per-slot meaning pending a per-scene consumer trace). Camera angle / face-target opcodes (motion-VM 0x38 / 0x4C) ported as StepResult::Pending placeholders pending the 12-bit fixed-point math.
