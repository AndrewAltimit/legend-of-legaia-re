Legend of Legaia - reverse engineering & clean-room engine port

Two coordinated tracks under one repo. Track 1: extract every asset on the disc, document every format with provenance back to a Ghidra function, build round-trip parsers. Track 2: a clean-room Rust engine port that consumes the extracted data, in the same legal pattern as ScummVM, OpenRCT2, OpenMW, and OpenLara. Bring your own disc image; the toolkit handles the rest.

Where we are

The disc is fully readable, every major subsystem is traced, all five runtime VMs plus the battle action state machine are ported, and the engine boots a real scene in a window or in the browser.

Asset preservation
Disc reader (Mode2/2352 + ISO9660), PROT.DAT TOC math, format detectors covering every documented PROT family (verified by a disc-gated coverage test). Round-trip parsers for TIM, TMD, VAB, SEQ, MES, ANM, MDT, LZS, plus the scene-bundle family. PSX memory-card walker round-trips Legaia save blocks. Formats →
Static analysis
Function dumps for the bulk of the binary, citation graph closed (every helper referenced in the docs has its own dump). All five runtime VMs decoded plus the battle action state machine. Every major SCUS subsystem traced. All major runtime overlays captured (title / town / battle / menu / world map / save / shop / level-up / minigame). Key functions →
Engine port
Clean-room Rust port. The runtime ticks the actor / field / move / motion / effect VMs against real PROT bytes, drives the battle action state machine through party + monster slots, resolves through the formula damage path, and assembles the field / battle / menu loop end-to-end. The legaia-engine shell boots a scene from PROT bytes alone, with a windowed renderer + dialog + BGM. Engine plan →
WASM viewer
The asset viewer compiles to wasm32-unknown-unknown and runs in this browser. Drop in a PROT.DAT or a full .bin disc image and it parses + renders client-side - no upload, no server. Try it →
