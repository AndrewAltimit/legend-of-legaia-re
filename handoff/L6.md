# Lane L6 handoff

## 1. `git stash` is shared across worktrees - it raced and lost work

`git stash` writes to a **single** `refs/stash` stack for the whole repository,
not per worktree. With ~10 lanes running concurrently, a
`git stash push` / `git stash pop` pair in one worktree is not atomic against
the others: between the push and the pop another lane can push (renumbering
`stash@{0}`) or pop (taking yours).

This happened here. Lane L6 ran `git stash push` / `git stash pop` around a
baseline test run and popped **lane `worktree-agent-a9e6a9e16b1604389`'s**
dev-menu WIP into the L6 worktree, while L6's own stash was taken elsewhere.
The stash list at the time already carried a `recovered: ... popped into the
wrong worktree by lane L2` entry, so at least two lanes hit this.

**Do not use `git stash` in a parallel-lane wave.** To get a clean tree for a
baseline measurement, commit to the lane branch and `git revert`/reset
afterwards, or copy the files aside.
