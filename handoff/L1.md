# Lane L1 (baka-fighter + muscle-dome) handoff

## URGENT for the dance / fishing / slot-machine lane

`git stash` is **repo-global across worktrees** - every lane shares one stash
stack. Lane L1 ran `git stash` / `git stash pop` once, and the pop took the
*other* lane's entry: the following working-tree changes landed in L1's
worktree instead of their author's, and L1's own two files ended up in the
other worktree (they were returned as a stash entry labelled
`recovered: ... popped into the wrong worktree by lane L2; restored`).

L1 has reverted the foreign changes out of its worktree and preserved them
verbatim here so nothing is lost:

- `handoff/L2-dance-fishing-slot.patch` - `git diff` over
  `crates/asset/src/minigame_slot_scene.rs`,
  `crates/engine-core/src/{dance.rs, fishing_chrome.rs, lib.rs, slot_machine.rs}`,
  `docs/subsystems/minigame-{dance,fishing}.md`.
  Re-apply with `git apply handoff/L2-dance-fishing-slot.patch` from the repo
  root of the owning worktree.
- `handoff/L2-dance_tutorial.rs.txt` -> `crates/engine-core/src/dance_tutorial.rs`
- `handoff/L2-minigame_floor.rs.txt` -> `crates/engine-core/src/minigame_floor.rs`

Both new modules need their `pub mod` lines back in
`crates/engine-core/src/lib.rs`; the patch carries them.

**No lane should use `git stash` while the wave is running.** Use a scratch
copy of the file, or `git diff > /tmp/...` plus `git checkout --`, instead.
