# Lane L11 handoff - two edits outside my owned paths

## 1. `docs/subsystems/script-vm.md` still calls `0x801F71E0` a real entry

Two places contradict what the PROT 0967 bytes say, and this lane has now
ignore-closed the row as `worklist_interior`:

- the triage table row `| 0x801F71E0 / 0x801F5748 | REAL, large | Per-actor
  command / queue loops ... |`
- the section `### The actor-band command loops (FUN_801F71E0 / FUN_801F5748)`,
  which credits `FUN_801F71E0` with 1070 instructions.

`0x801F71E0` is not an entry in any based image. In the extracted PROT 0967
image at base `0x801F69D8` the word at the VA is `lw v1,-0x42dc(v0)` and the
word before it is `lui v0,0x8008` - one address load split across the address -
and the VA is the target of `bne v1,v0,0x801f71e0` sixteen bytes earlier.
Between it and the only prologue below it (`0x801F6B78`, `addiu sp,sp,-0x38`)
there is no `jr ra`; the enclosing routine's sole `jr ra` is `0x801F7474`.
`locate-entry-image.py` reports no frame and no in-image `jal` across all 31
based images.

The "1070 instructions" figure is separately unattributable: it comes from
`overlay_0897_801f71e0`, a dump whose bytes match no extracted image at the
queried VA. The base-correct dump `overlay_effect_0967_801f71e0` carries 167
instructions - the routine's tail, printed from the label onward, because
`dump_effect_overlay_0967.py` creates a function at each traced hit address.

Corrected text now lives in `docs/reference/functions.md` under
`### 801F71E0 is a label, not an entry`, and the ignore reason is in
`scripts/ci/port-catalog-ignore.toml`. `script-vm.md` should point at that
section instead of restating the entry claim. `FUN_801F5748` is untouched by
this finding.

## 2. `crates/engine-vm/README.md` does not list the two new modules

- `world_map_panel_actors` - the seven `ctx[+0x54]` panel actor state machines
  in the PROT 0897 world-map band plus the field party HUD.
- `panel_backread_loader` - the slot-B "FIELD BACK READ" staged-loader tick
  (`FUN_801F6B24`, PROT 0978). The `panel_*` filename follows this lane's
  owned-path grant; the module's subject is a texture streamer, not a panel,
  so a rename to `field_backread_loader` would read better if another lane is
  already touching that file.
